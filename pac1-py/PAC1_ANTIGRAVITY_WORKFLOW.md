# Инструкция по выполнению и трекингу задач (t01 - t40)

## Роли и режим
- Режим: `LLM_PROVIDER=claude`
- Моя роль: Я выступаю "мозгом" анализа уулчшения системы после просмотра логово по задаче. 

## Цикл решения одной задачи (tXX)
1. **Запуск**: Выполняется фоновая команда `python main.py tXX`.
2. **Исполнение**: Агент решает задачу.
3. **Оценка результата**: Фиксируется финальный вывод (benchmark score).
4. **Анализ (при ошибках)**: Если `Score < 1.00`:
   - Анализирую, где СИСТЕМА (мой ответ) разошлась с ожиданиями бенчмарка.
   - Заношу выводы о собственных ошибках в сводку.
   - Корректирую свой логический аппарат или подход, соблюдая все принципы из `benchmark_system_questions_answers.md`.
   ВАЖНО: не подстраиваться под частные случаи и частные значения задачи. Система должна быть универсальной для решения любых задач из бенчмарка.
   - Перезапускаю задачу, пока не будет `1.00`.
