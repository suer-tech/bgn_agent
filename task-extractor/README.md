# BitGN Извлечёнщик Задач

Извлекает задачи из бенчмарков BitGN с полными данными рабочего пространства.

## Что делает скрипт

Для каждой задачи в бенчмарке скрипт:
1. Подключается к BitGN API
2. Получает информацию о бенчмарке и список задач
3. Запускает пробную сессию для каждой задачи
4. Исследует рабочее пространство runtime (структуру каталогов и файлы)
5. Сохраняет все данные по задачам в отдельные папки

## Структура вывода

```
extracted_tasks/
└── bitgn_sandbox/
    ├── benchmark_info.json
    ├── t01/
    │   ├── instruction.txt        # Вопрос/инструкция задачи
    │   ├── workspace_tree.md      # Визуализация структуры каталогов
    │   ├── metadata.json          # Полные метаданные задачи
    │   └── files/                 # Все файлы рабочего пространства
    │       ├── AGENTS.MD
    │       ├── contacts/
    │       │   └── alice.md
    │       └── notes/
    │           └── todo.md
    ├── t02/
    │   └── ...
    └── t03/
        └── ...
```

## Использование

### Установка зависимостей

```bash
cd task-extractor
uv sync
```

### Извлечение всех задач из sandbox бенчмарка

```bash
uv run python main.py
```

### Извлечение конкретных задач

```bash
uv run python main.py bitgn/sandbox --tasks t01 t02
```

### Извлечение из PAC1 бенчмарка

```bash
uv run python main.py bitgn/pac1-dev
```

### Параметры

- `benchmark`: ID бенчмарка (по умолчанию: `bitgn/sandbox`)
- `--tasks`: Конкретные ID задач для извлечения (по умолчанию: все)
- `--output`: Выходная директория (по умолчанию: `extracted_tasks`)
- `--depth`: Максимальная глубина обхода каталогов (по умолчанию: 3)

## Переменные окружения

Создайте файл `.env` или установите следующие переменные окружения:

```bash
BENCHMARK_HOST=https://api.bitgn.com
OUTPUT_DIR=extracted_tasks
```

## Поддерживаемые Runtime

- **Mini Runtime**: Для sandbox бенчмарка
- **PCM Runtime**: Для PAC1 бенчмарка

Скрипт автоматически определяет тип runtime и использует соответствующий API.
